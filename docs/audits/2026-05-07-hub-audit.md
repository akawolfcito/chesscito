# Hub Audit — 2026-05-07

> **Findings**: the new hub is a pure-presentation skin. Every CTA delegates to the legacy `<PlayHubRoot>` via `?legacy=1` URL bounce, which is the root cause of the brokenness the user reported on smoke. Coach has zero entry points. Mastery tiles all collapse to one destination. The "Play" button bypasses the new arena scaffold.

---

## Architecture as-is

```
URL                        Renders                        Heavy logic
───────                    ─────────                       ─────────
/hub                       <HubScaffoldClient>             pure presentation
/hub?legacy=1              <PlayHubRoot>                   sheets, on-chain, mutations
/hub?legacy=1&action=X     <PlayHubRoot> seeded            opens sheet X on mount
/play-hub                  rewrites to / → <PlayHubRoot>   same as above
```

**Critical**: new hub `<HubScaffold>` has NO direct sheets, NO direct routes. Each tap pushes `/hub?legacy=1&action=X`, the legacy mounts, opens sheet X, user closes, useEffect bounces back to `/hub`. **Every interaction is a 2-page round trip**, which is what the user perceives as "broken".

## B2 — "Pass Training Play in Arena stuck" — root cause

User reports: tap 26d chip → opens ProSheet → tap "Play in Arena" → nothing happens / returns to hub.

Trace:
1. `/hub` (scaffold) → tap chip → `router.push("/hub?legacy=1&action=pro")`
2. Server renders `<PlayHubRoot initialAction="pro">` → `proSheetOpen = true`
3. `<ProSheet>` renders with `<ProActiveCTA>` — captures `source = "/hub"` (frozen)
4. User taps "Play in Arena" → `<ProActiveCTA.handleClick>`:
   - `router.push("/arena")`  ← async, not yet committed
   - `onClose()` → `setProSheetOpen(false)`  ← synchronous
5. **`useEffect` at `play-hub-root:215`** detects `proSheetOpen` flipped + `initialAction === "pro"` → bounces `router.push("/hub")` ← **races with the `/arena` push**
6. The bounce wins. User sees `/hub` (scaffold). **Effectively, "Play in Arena" navigates to hub.**

**Fix**: make `<ProActiveCTA>` await navigation OR have the legacy bounce-back effect skip when a deliberate non-bounce navigation is in flight. Cleanest: `<ProSheet>` direct from new hub (no legacy round-trip) so there's no bounce loop.

## B5 — "Play button → legacy /play-hub" — root cause

`hub-scaffold-client.tsx:267-270`:
```ts
onPlayPress={() => {
  track("hub_play_tap");
  router.push(LEGACY_HUB);  // = "/hub?legacy=1"
}}
```

The PLAY CTA should go to `/arena` (which now defaults to the scaffold view per `af160bb`). Instead it goes to the legacy hub which renders `<PlayHubRoot>` — the user has to tap a *second* control to reach Arena.

**Fix**: change `onPlayPress` to `router.push("/arena")` directly.

## /play-hub legacy alias status

`apps/web/next.config.js` already rewrites `/play-hub → /`. The "/" page renders `<PlayHubRoot>`. So `/play-hub` is a working alias to legacy.

**Decision needed**: do we kill `/play-hub` entirely? Looking at usage:
- `next.config.js` rewrite → `/`
- No code references `/play-hub` anywhere in `src/` (verified via grep)
- External links (Twitter, MiniPay registration) may link to `/play-hub`
- Memory mentions `MEMORY.md` refers to `/play-hub` historically

**Recommendation**: kill the rewrite + the legacy hub flow once we wire all CTAs directly. Until then, keep alias as safety net.

## Path to unification

Two strategies:

### Strategy A — port handlers to scaffold (recommended)
- Copy each sheet wiring from `<PlayHubRoot>` into `<HubScaffoldClient>` directly
- Open sheets in-place via React state, no URL bouncing
- Sheets stay (TrophiesSheet, ProSheet, ShopSheet, BadgeSheet)
- Legacy `<PlayHubRoot>` becomes dead code → delete in next pass
- Effort: ~1-2 days. Each sheet is independent.

### Strategy B — replace sheets with dedicated routes
- `/trophies`, `/shop`, `/badges/[piece]`, `/pro` as full routes (some already exist)
- Hub CTAs use `router.push` to those routes (no sheet pattern)
- More refactor; better deep-linking; cleaner mental model
- Effort: ~3-4 days

**My pick: Strategy A** — faster, lower risk, sheets already exist & tested.
