# Chesscito PRO — Fase 0 Implementation Plan

**Date**: 2026-04-29
**Goal**: Time-to-revenue. Activar primer SKU comercial real (PRO mensual) reutilizando el flujo Shop + Coach existente. **Sin** redeploy de contratos.

## Decisiones (confirmadas 2026-04-29)
- itemId on-chain: **6** ✅
- Precio: **$1.99 / mes** (priceUsd6 = `1_990_000`) ✅
- Duración: **30 días** ✅
- Renovación: **manual** en v1 (usuario re-compra) ✅
- Canal inicial: **Web**, MiniPay-safe ✅
- Pago: **stablecoin only** (USDC) — defense-in-depth ✅
- **Beneficios v1**: SOLO bypass de Coach credits. No Arena, no achievements, no VictoryNFT gratis, no cooldowns, no badges. Otros beneficios = copy de roadmap (Fase 1+). ✅

- **Telemetría**: Supabase events (no console-only). Eventos: `pro_card_viewed`, `pro_cta_clicked`, `pro_purchase_started`, `pro_purchase_confirmed`, `pro_purchase_failed`, `coach_pro_bypass_used`. Reusar helper si existe; si no, crear integración mínima no bloqueante. ✅
- **`setItem(6)` on-chain**: coordinar al final del release. Documentar en release checklist: `ShopUpgradeable.setItem(6, 1_990_000, true)` en Celo Mainnet con admin wallet. ✅

## Riesgos transversales
1. **No-redeploy**: depende de que admin llame `setItem(6, 1_990_000, true)` antes del release. **Bloqueante** para release final, no para merge.
2. **In-memory rate limit**: ya conocido (cold-start reset). Aceptable para v1.
3. **Self-extension del PRO**: el atacante no puede extender sin pagar (verify-pro valida tx). Riesgo real: replay con tx ajenos → bloqueado por filtro `buyer === wallet`.
4. **MiniPay UX**: PRO requiere approve+buy igual que packs Coach. Probado.
5. **Renovación manual**: si el usuario olvida, PRO expira sin warning. Mitigación: banner "Expira en X días" cuando `expiresAt - now < 5d` (deferible a Fase 0.1).
