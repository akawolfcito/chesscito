# Red Team Report — 2026-04-05

Post UX-audit fixes regression check. 3 parallel agents audited all flows.

---

## Broken Flows (7 High)

### H1. VictoryClaiming back button permanently disabled
- **File:** `components/arena/victory-claiming.tsx:118`
- **Description:** `claimStep` defaults to `"signing"` but `ArenaEndState` never passes it. Back button is `disabled={claimStep !== "done"}` — always disabled. If claim tx hangs, user is trapped.
- **Reproduce:** Win, tap "Claim Victory", observe claiming overlay. Back button is disabled with no way out except page refresh.

### H2. Restored "claiming" phase from sessionStorage creates dead end
- **File:** `app/arena/page.tsx:105-109`
- **Description:** On reload mid-claim, `claimPhase="claiming"` is restored but game state is `"selecting"`. `ArenaEndState` never renders (not end state), leaving stale claiming phase with no visible effect or code path to clear it.
- **Reproduce:** Start a claim, immediately refresh page while deadline is valid.

### H3. Wallet disconnect mid-receipt hangs forever
- **File:** `app/arena/page.tsx:327,353`
- **Description:** If wallet disconnects during `waitForTransactionReceipt`, the Promise may hang indefinitely. User stuck in VictoryClaiming overlay with permanently disabled back button.
- **Reproduce:** Start claim, approve token, disconnect wallet while waiting for mint receipt.

### H4. Shop sheet stays open under Purchase Confirm sheet
- **File:** `app/page.tsx:789-793`
- **Description:** Tapping "Buy" sets `confirmOpen(true)` but never sets `storeOpen(false)`. Both sheets mount at z-50 simultaneously. Dismissing confirm reveals shop still open.
- **Reproduce:** Open Shop, tap "Buy" on enabled item. Both sheets visible.

### H5. Cancel button clickable during active transaction
- **File:** `components/play-hub/purchase-confirm-sheet.tsx:97-104`
- **Description:** Cancel button disabled only by `isWriting`, but there's a window between approve confirm and buy submit where `isWriting` is false and `purchasePhase` is `"buying"`. Cancel closes sheet while async function continues.
- **Reproduce:** Start purchase, after approve tx confirms, quickly tap "Cancel" before buy tx is sent.

### H6. showPieceComplete not reset when switching pieces
- **File:** `app/page.tsx:735-744`
- **Description:** Piece rail handler resets `showBadgeEarned` but NOT `showPieceComplete`. If overlay is visible and user taps another piece, it persists with stale data.
- **Reproduce:** Complete last rook exercise, wait for PieceCompletePrompt, tap "Bishop" in rail.

### H7. handleSubmitScore has no concurrency guard
- **File:** `app/page.tsx:557-601`
- **Description:** Unlike `handleClaimBadge` (checks `isClaimBusy`), `handleSubmitScore` has no guard. Rapid double-tap can fire two `writeContractAsync` calls before React disables the button.
- **Reproduce:** Rapidly double-tap "Submit Score" CTA on a slow device.

## Dead Ends (2 Medium)

### M1. Failure phase + disconnected wallet = no CTA for 1.5s
- **File:** `lib/game/context-action.ts:21-23`
- **Description:** When `phase==="failure"` and wallet not connected, `getContextAction` returns null. Board is locked. Only the 1.5s auto-reset timer saves the user.
- **Reproduce:** Disconnect wallet, fail a single-move exercise.

### M2. Both showPieceComplete and showBadgeEarned could stack
- **File:** `app/page.tsx:860-893`
- **Description:** Both are z-60 overlays with no mutual exclusion. Current flow prevents both being true simultaneously, but no guard exists.
- **Reproduce:** Difficult — would require React batching edge case.

## State Leaks (4 Medium)

### M3. Badge auto-dismiss timer differs from "Later" button
- **File:** `app/page.tsx:454-458 vs 500-503`
- **Description:** Timer calls `resetBoard()` silently. "Later" button calls `setShowPieceComplete(true)`. Countdown bar implies they do the same thing — they don't.
- **Reproduce:** Earn badge, let 15s timer expire vs. clicking "Later". Different outcomes.

### M4. Coach analysis state leaks into next game
- **File:** `app/arena/page.tsx:412-428`
- **Description:** `coachCredits` not reset by `resetArenaState`. In-flight fetch from `startCoachAnalysis` has no abort signal — callbacks can fire after reset.
- **Reproduce:** Start coach analysis, quickly tap "Play Again" while loading.

### M5. sessionStorage claim data persists across games
- **File:** `app/arena/page.tsx:74-88`
- **Description:** Restored `claimPhase="success"` has no visible effect because game state is `"selecting"` on fresh mount. Stale data never cleaned up.
- **Reproduce:** Win, claim, share to WhatsApp, return. Claim success overlay won't appear.

### M6. elapsedMs/moves only track last exercise
- **File:** `app/page.tsx:119,438,569`
- **Description:** `timeMs` submits `1000n` when phase is not "success". On-chain time value is architecturally meaningless for score submissions after board reset.
- **Reproduce:** Complete all exercises, wait for auto-reset, submit score. `timeMs = 1000n`.

## Race Conditions (3 Medium)

### M7. Preparing timeout not cleaned up on unmount
- **File:** `app/arena/page.tsx:436-441`
- **Description:** 400ms `setTimeout` not stored in ref, never cleaned up. Navigating away during preparing fires `game.startGame()` on unmounted component.
- **Reproduce:** Tap difficulty, navigate away within 400ms.

### M8. isWriting shared across all three transaction flows
- **File:** `app/page.tsx:114,387-388`
- **Description:** Single `useWriteContract()` hook means `isWriting` blocks unrelated flows. Claim silently blocked during unrelated purchase.
- **Reproduce:** Submit score, then quickly try to claim badge before submit tx is signed.

### M9. Sheet backdrop swipe-to-dismiss not guarded during transaction
- **File:** `components/play-hub/purchase-confirm-sheet.tsx:41`
- **Description:** Sheet can be dismissed via backdrop tap or swipe even during active tx.
- **Reproduce:** During "Approving..." state, tap dark backdrop area.
